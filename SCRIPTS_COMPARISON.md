# 📊 Порівняння скриптів: RUN_FULL.sh vs DEPLOY.sh

## 🎯 Коротка відповідь

| Аспект                   | RUN_FULL.sh    | DEPLOY.sh                         |
| ------------------------ | -------------- | --------------------------------- |
| **Тип**                  | 🚀 ЗАПУСК       | 📦 РОЗГОРТАННЯ + ЗАПУСК            |
| **Перевірки**            | ❌ Ні           | ✅ Так (Python, залежності, файли) |
| **Встановлення**         | ❌ Ні           | ✅ Так (pip install)               |
| **Налаштування**         | ❌ Ні           | ✅ Так (конфіг, workflows)         |
| **Запуск**               | ✅ Так          | ✅ Так                             |
| **Коли використовувати** | Вже розгорнуто | Перший раз / нова машина          |

---

## 📋 Детальне порівняння

### RUN_FULL.sh - 🚀 ЗАПУСК (Hyper-Power System)

**Призначення**: Запустити вже розгорнену систему

**Що робить**:
```
1. Очищає старі процеси
2. Запускає Enhanced Analyzer (5 шарів)
3. Запускає MCP Server (16 інструментів)
4. Постійно моніторить процеси
```

**Передумови**:
- ✅ Система вже розгорнута
- ✅ Залежності встановлені
- ✅ Конфігурація готова

**Час виконання**: ~5-10 секунд

**Результат**:
```
✅ Enhanced Analyzer запущено (PID: XXXXX)
✅ MCP Server запущено (PID: XXXXX)
✅ Система готова до роботи
```

**Коли використовувати**:
- Кожен день після перезавантаження
- Коли система була зупинена
- Для швидкого запуску

---

### DEPLOY.sh - 📦 РОЗГОРТАННЯ + ЗАПУСК (Advanced System v2.0)

**Призначення**: Повне розгортання + запуск системи

**Що робить**:
```
КРОК 1: ПЕРЕВІРКИ
  - Перевіряє Python 3.x
  - Перевіряє pip
  - Перевіряє файли проекту
  - Перевіряє workflows
  - Завантажує конфігурацію

КРОК 2: ВСТАНОВЛЕННЯ
  - Встановлює залежності (pip install)
  - Перевіряє залежності

КРОК 3: НАЛАШТУВАННЯ
  - Створює директорії для звітів
  - Оновлює версію workflows
  - Налаштовує Windsurf
  - Встановлює pre-commit hook

КРОК 4: ПЕРШИЙ АНАЛІЗ
  - Запускає перший аналіз проекту
  - Перевіряє звіти

КРОК 5: MCP СЕРВЕР
  - Запускає MCP сервер
  - Синхронізує аналіз з memory

КРОК 6: ЗАПУСК
  - Запускає watch mode
  - Постійне оновлення звітів
```

**Передумови**:
- ❌ Система може бути не розгорнута
- ❌ Залежності можуть бути не встановлені
- ❌ Конфігурація може бути не готова

**Час виконання**: ~2-5 хвилин (перший раз)

**Результат**:
```
✅ Python перевірено
✅ Залежності встановлені
✅ Конфігурація готова
✅ Перший аналіз завершено
✅ MCP сервер запущено
✅ Watch mode активний
```

**Коли використовувати**:
- Перший раз на новій машині
- Після оновлення системи
- Коли потрібна повна переналаштування
- Коли щось зламалось

## 🎯 Рекомендована послідовність

### Сценарій 1: Перший раз на новій машині

```
1️⃣ DEPLOY.sh (розгортання + налаштування)
   └─ Встановлює все необхідне
   
2️⃣ Перезавантажити Windsurf
   └─ Щоб MCP конфіг був активний
   
3️⃣ RUN_FULL.sh (запуск)
   └─ Для наступних запусків
```

### Сценарій 2: Кожен день

```
1️⃣ RUN_FULL.sh (швидкий запуск)
   └─ Займає 5-10 секунд
   
2️⃣ Перезавантажити Windsurf
   └─ Щоб підключитися до MCP
   
3️⃣ Використовувати Cascade
   └─ З 16 інструментами
```

### Сценарій 3: Щось зламалось

```
1️⃣ DEPLOY.sh (повна переналаштування)
   └─ Перевіряє все і переналаштовує
   
2️⃣ Перезавантажити Windsurf
   
3️⃣ RUN_FULL.sh (запуск)
```

## 📊 Таблиця вибору

| Ситуація             | Вибір       | Причина                     |
| -------------------- | ----------- | --------------------------- |
| **Перший раз**       | DEPLOY.sh   | Потрібна повна налаштування |
| **Кожен день**       | RUN_FULL.sh | Швидко, все вже готово      |
| **Після оновлення**  | DEPLOY.sh   | Потрібна переналаштування   |
| **Щось не працює**   | DEPLOY.sh   | Перевіряє все заново        |
| **Просто запустити** | RUN_FULL.sh | Швидкий запуск              |
| **Нова машина**      | DEPLOY.sh   | Потрібна повна налаштування |
